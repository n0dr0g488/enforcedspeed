3A. TICKET IMAGE SUBMISSIONS

3A.1 Upload and Automated Image Processing

Ticket Image uploads are analyzed to verify the information entered by the User regarding their ticket. Below is a technical description of the Ticket Image upload and analysis process. Company provides this description as a good faith account of how Users' Ticket Images are processed.

When a User submits information about a speeding ticket, the User may optionally upload an image of the ticket (i.e., a Ticket Image). Under normal operation, image handling functions as follows:

Sanitization and Standardization. Uploaded images are sanitized server-side and re-encoded or resized to remove embedded metadata and prepare the image for Automated Processing.

Temporary Quarantine Storage. Sanitized images are placed in private, non-public storage for short-term processing. Only a reference to the image is stored in application databases.

AI-Based Image Verification. Background processing systems retrieve the image and submit it to an AI-based image analysis service (currently Google Gemini) to determine whether the image is a genuine traffic citation and to extract relevant ticket values (i.e., speed limit, ticketed speed). The AI model independently assesses the document before any verification is applied. The AI model may extract any information from the Ticket Image that is relevant to the operation, verification, improvement, or security of the Services, including but not limited to: document type classification, speed values, jurisdiction indicators, date information, and document authenticity markers. The AI model is not specifically instructed to extract names, dates of birth, driver's license numbers, or Social Security numbers, although Company cannot guarantee that such information is never incidentally processed during image analysis.

Deletion After Processing. Following completion of Automated Processing, the system is designed to delete the image as part of cleanup procedures.

Deletion Backstop. Temporary image storage is subject to automated lifecycle controls intended to delete images within a limited time window (not to exceed twenty-four (24) hours), even if processing is not completed as expected.
