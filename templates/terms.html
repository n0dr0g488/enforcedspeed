3. TICKET IMAGE SUBMISSIONS

3.1 Upload and Automated Image Processing

Ticket Image uploads are analyzed to verify the information entered by the User regarding their ticket. Below is a technical description of the Ticket Image upload and analysis process. Company provides this description as a good faith account of how Users’ Ticket Images are processed. This description is intended to be comprehensible to the average user and is not intended as, and should not be construed to be, a binding specification of the Services.

When a User submits information about a speeding ticket, the User may optionally upload an image of the ticket (i.e., a Ticket Image). Under normal operation, image handling is functions as follows:

Sanitization and Standardization. Uploaded images are sanitized server-side and re-encoded or resized to remove embedded metadata and prepare the image for Automated Processing.

Temporary Quarantine Storage. Sanitized images are placed in private, non-public storage for short-term processing. Only a reference to the image is stored in application databases.

Automated OCR Processing. Background processing systems retrieve the image and submit it to automated OCR services to extract text and identify relevant ticket values (i.e., speed, speed limit, location).

Deletion After Processing. Following completion of Automated Processing, the system is designed to delete the image as part of cleanup procedures.

Deletion Backstop. Temporary image storage is subject to automated lifecycle controls intended to delete images within a limited time window (not to exceed twenty-four (24) hours), even if processing is not completed as expected.
