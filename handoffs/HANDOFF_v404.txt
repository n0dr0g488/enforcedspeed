Goal of v404 (single scoped change)
Stop the Home feed from snapping back to the top after Like / Comment / Reply / Delete-comment actions.

Root cause
- These actions are standard POST forms that redirect back to the Home feed.
- After redirect, the browser resets scroll position to the top.
- Additionally, the Enter-to-submit helper used form.submit(), which bypasses submit event listeners.

What changed
- templates/home_feed.html
  - Added id="ticket-{{ r.id }}" to each ticket card so we can restore to the exact card.
  - Added a small scroll-preservation helper:
    - On submit of Like / Add Comment / Delete Comment forms, store scroll + card id in sessionStorage.
    - On page load, restore scroll (prefer card scrollIntoView).
  - Updated the Enter-to-submit helper to:
    - Save scroll state first
    - Use requestSubmit() when available (fallback to submit())

Files changed
- templates/home_feed.html
- VERSION.txt

- handoffs/HANDOFF_v404.txt
